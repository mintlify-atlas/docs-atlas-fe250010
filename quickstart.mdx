---
title: 'Get started with Kinoko'
description: 'Build and run the Kinoko MapleStory v95 server emulator from source using Java 21 and Maven, then connect with a compatible client.'
---

This page walks you through building Kinoko from source and running it locally. You need Java 21, Maven, and a set of WZ data files extracted from a v95 MapleStory client. No external database setup is required — Kinoko uses SQLite by default.

<Note>
  Kinoko requires **Java 21**. Earlier versions of the JDK are not supported. Verify your installed version with `java -version` before proceeding.
</Note>

<Steps>
  <Step title="Clone the repository">
    Clone the Kinoko source code to your local machine.

    ```bash
    git clone https://github.com/iw2d/kinoko.git
    cd kinoko
    ```
  </Step>

  <Step title="Place WZ files in the wz/ directory">
    Kinoko reads game data from WZ files extracted from a v95 MapleStory client. Copy all of the following files into the `wz/` directory at the root of the repository before building:

    ```
    wz/
    ├── Character.wz
    ├── Item.wz
    ├── Skill.wz
    ├── Morph.wz
    ├── Map.wz
    ├── Mob.wz
    ├── Npc.wz
    ├── Reactor.wz
    ├── Quest.wz
    ├── String.wz
    └── Etc.wz
    ```

    The server will not start if any of these files are missing or placed in a different directory.
  </Step>

  <Step title="Build with Maven">
    Compile and package the server into a single executable JAR.

    ```bash
    mvn clean package
    ```

    Maven produces `target/server.jar` — a fat JAR that includes all runtime dependencies.
  </Step>

  <Step title="Run the server">
    Start Kinoko with the packaged JAR.

    ```bash
    java -jar target/server.jar
    ```

    On first startup, the server loads all WZ providers and initializes the SQLite database under `data/`. Provider loading can take several seconds depending on hardware.

    <Note>
      Kinoko uses **SQLite** as the default database provider. No installation or configuration is required — the database file is created automatically in the `data/` directory on first run.
    </Note>
  </Step>

  <Step title="Connect with a v95 MapleStory client">
    Point your v95 MapleStory client at the login server.

    ```
    Host: 127.0.0.1
    Port: 8484
    ```

    With the default configuration, `AUTO_CREATE_ACCOUNT` is enabled, so logging in with any username automatically creates a new account. A secondary password (PIC) is also required by default.
  </Step>
</Steps>
